# Методы Subscribe API | Payme Business

к сведению

Методы Subscribe API — независимы и работают по отдельности.

Для работы с пластиковыми картами в Subscribe API используется 2 вида методов.

**Методы для клиентской части приложения мерчанта:**

-   [cards.create. Создание токена пластиковой карты](https://developer.help.paycom.uz/metody-subscribe-api/cards.create).
    
-   [cards.get\_verify\_code. Запрос кода для верификации карты](https://developer.help.paycom.uz/metody-subscribe-api/cards.get_verify_code).
    
-   [cards.verify. Верификация карты с помощью кода отправленного по СМС](https://developer.help.paycom.uz/metody-subscribe-api/cards.verify).
    

**Методы для серверной части приложения мерчанта:**

-   [cards.check. Проверка токена карты](https://developer.help.paycom.uz/metody-subscribe-api/cards.check).
    
-   [cards.remove. Удаление токена карты](https://developer.help.paycom.uz/metody-subscribe-api/cards.remove).
    

Для работы с чеком, в Subscribe API используются методы для серверной части приложения мерчанта:

-   [receipts.create. Создание чека на оплату](https://developer.help.paycom.uz/metody-subscribe-api/receipts.create).

-   [receipts.pay. Оплата чека](https://developer.help.paycom.uz/metody-subscribe-api/receipts.pay).
-   [receipts.send. Отправка инвойса](https://developer.help.paycom.uz/metody-subscribe-api/receipts.send).

-   [receipts.cancel. Установка оплаченного чека в очередь на отмену](https://developer.help.paycom.uz/metody-subscribe-api/receipts.cancel).
-   [receipts.check. Проверка статуса чека](https://developer.help.paycom.uz/metody-subscribe-api/receipts.check).

-   [receipts.get. Полная информация по чеку](https://developer.help.paycom.uz/metody-subscribe-api/receipts.get).
-   [receipts.get\_all. Полная информация по чекам за определенный период](https://developer.help.paycom.uz/metody-subscribe-api/receipts.get_all).
