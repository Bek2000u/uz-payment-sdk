# Взаимодействие протоколов Merchant API и Subscribe API | Payme Business

Порядок взаимодействия протоколов Merchant API и Subscribe API осуществляется в соответствии с процессом, показанным на схеме:

![](https://cdn.payme.uz/help/payme-business-help-developer/ru/api_interaction/api_interaction.jpg#pw)

Каждый протокол тестируется отдельно.

**Тестирование Merchant API**

Протокол Merchant API можно протестировать в [Песочнице](https://developer.help.paycom.uz/pesochnitsa).

**Тестирование Subscribe API**

Для тестирования Subscribe API:

1.  Войдите в [тестовый кабинет мерчанта](https://merchant.test.paycom.uz/#/login).

Логин для входа в кабинет: номер вашего сотового телефона

Пароль для входа в кабинет: qwerty

SMS-код: 666666

2.  В тестовом кабинете создайте бизнес.
3.  Добавьте в бизнес кассу «Виртуальный терминал».
4.  Ключ-пароль от кассы запросите у технического специалиста Payme Business.
5.  Тестовые запросы посылаются на тестовую платформу:

[https://checkout.test.paycom.uz/api](https://checkout.test.paycom.uz/api)
